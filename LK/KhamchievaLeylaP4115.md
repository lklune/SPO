# Лабораторная работа №3

**Тема:** Реализация формирования линейного кода в терминах набора инструкций абстрактной виртуальной машины

## Цель работы

Реализовать генерацию линейного кода по графу потока управления для абстрактной виртуальной машины, а также обеспечить экспорт результата в ассемблерный листинг, пригодный для последующей сборки и запуска.

## Краткое описание проекта

В работе реализован конвейер:

1. разбор исходного текста программы;
2. построение AST;
3. построение CFG по AST;
4. генерация линейного кода;
5. экспорт линейного кода в текстовый asm-файл;
6. сборка и запуск результата в RemoteTasks.

Дополнительно реализован тестовый пример программы-калькулятора с интерактивным вводом операторов и чисел, включая вычисление чисел Фибоначчи.

## Основные структуры данных

В проекте используются следующие основные структуры:

1. `Node`  
   Узел дерева разбора/AST.

2. `Operation`  
   Внутреннее представление операций для CFG и codegen.

3. `BasicBlock`  
   Базовый блок графа потока управления.

4. `CFG`  
   Граф потока управления функции.

5. `FunctionSignature`, `FunctionArg`, `Function`  
   Описание сигнатуры функции, аргументов и самой функции.

6. `Instruction`, `Operand`, `LinearCode`  
   Представление линейного кода.

7. `RegisterAllocator`, `VariableBinding`  
   Структуры для связывания переменных с памятью и временными регистрами.

8. `CompiledFunction`, `CompiledFunctionCollection`  
   Представление результата генерации кода для одной или нескольких функций.

## Организация генерации кода

Генерация кода построена вокруг двух этапов:

### 1. Построение CFG

В файле `src/cfg.c` AST преобразуется в граф потока управления.

Поддерживаются:

- последовательности инструкций;
- объявления переменных;
- присваивания;
- условные операторы `if`;
- циклы `while` / `until`;
- оператор `break`;
- вызовы функций.

Для каждого базового блока сохраняется список операций и связи с последующими блоками.

### 2. Генерация линейного кода

В файле `src/codegen.c` CFG преобразуется в линейный код.

Используется:

- генерация меток для блоков;
- преобразование выражений в инструкции;
- генерация сравнений и переходов;
- генерация вызовов функций;
- размещение переменных в памяти.

## Особенности реализации

В данной реализации есть несколько важных особенностей:

1. Используется собственная VM с банками памяти:
   - `code_ram`
   - `constants_ram`
   - `data_ram`
   - `stack_ram`

2. Для запуска в интерактивном режиме используются специальные скрытые регистры:
   - `sin` для ввода
   - `sout` для вывода

3. Для работы с терминалом добавлены runtime-подпрограммы:
   - `readByte`
   - `writeByte`
   - `readOp`
   - `readDigit`
   - `writeInt`

4. В проекте отдельно отлаживался интерактивный калькулятор, который:
   - начинает с `0`;
   - принимает оператор и число;
   - пересчитывает накопленный результат;
   - поддерживает `+`, `-`, `*`, `/`, `f`, `q`.

## Генерация программы калькулятора

Файл `calc.src` используется как демонстрационная программа.

Логика программы:

- в `main` хранится текущее значение аккумулятора `acc`;
- вводится оператор `op`;
- вводится цифра `digit`;
- вычисляется числовой операнд `operand = digit - 48`;
- в зависимости от кода оператора изменяется `acc`;
- результат выводится через `writeInt`.

Коды операторов задаются ASCII-значениями:

- `43` — `+`
- `45` — `-`
- `42` — `*`
- `47` — `/`
- `102` — `f`
- `113` — `q`
- `48` — `'0'`

Это связано с тем, что ввод в VM организован побайтово.

## Пример программы

Фрагмент `calc.src`:

```txt
def main()
    int acc;
    byte op;
    byte digit;
    int operand;

    acc = 0;
    writeInt(acc);
    writeByte(10);
    op = readOp();

    while op != 113
        digit = readDigit();
        operand = digit - 48;

        if op == 43 than
        {
            acc = acc + operand;
        }

        if op == 102 than
        {
            acc = fib(operand);
        }

        writeInt(acc);
        writeByte(10);
        op = readOp();
    end
end
```

Функция Фибоначчи:

```txt
def fib(n of int)
    int r;

    r = 1;

    if n > 2 than
    {
        r = fib(n - 2) + fib(n - 1);
    }
end
```

## Получаемый результат

После генерации компилятор создает:

- `generated/main.linear_code.txt`
- `generated/fib.linear_code.txt`
- `generated/program.asm`

Общий файл `program.asm` содержит:

1. заголовок секции:

```asm
[section code_ram]
    JMP __start
```

2. стартовую точку:

```asm
__start:
    MOV 1048576, r3
    MOV r3, sp
    CALL main
    END
```

3. код пользовательских функций;
4. runtime-подпрограммы ввода-вывода.

## Тестирование

Для сборки и генерации использовались команды:

```bash
bison -d parser/parser.y -o parser/parser.tab.c
flex -o parser/lex.yy.c parser/lexems.l
make clean
make calc
```

Для сборки и запуска в RemoteTasks использовались команды:

```bash
export id=$(tt -w -id -s Assemble asmListing generated/program.asm definitionFile vm.target.pdsl archName vm)
tt -g "$id" -r stderr.txt
tt -g "$id" -r stdout.txt
tt -g "$id" -r out.ptptb > out.ptptb
```

Интерактивный запуск:

```bash
tt -il -s ExecuteBinaryWithInteractiveInput definitionFile vm.target.pdsl stdinRegStName sin stdoutRegStName sout archName vm binaryFileToRun out.ptptb ipRegStorageName ipst finishMnemonicName END codeRamBankName code_ram
```

```text
1

8

8
q
```
